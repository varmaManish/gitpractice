🚀 Git & GitHub Full Guide (Detailed & Interactive)

📦 1. Install Git

To start using Git, you need to install it on your computer. Visit git-scm.com/downloads and download the version suitable for your OS. Install it with default settings.

🔧 2. Configure Git

Set your name and email so Git can label your commits:

git config --global user.name "Your Name"
git config --global user.email "you@example.com"
📁 3. Initialize a Git Repository

Navigate to your project folder and initialize Git:

cd path/to/your/project
git init

This creates a hidden .git folder that tracks changes.

📤 4. Add and Commit Files

Stage your files and commit them with a message:

git add .
git commit -m "Initial commit"

git add . adds all files. You can also add specific files like git add index.html.

🌐 5. Connect to GitHub

Create a new repository on GitHub (don’t initialize with README). Then connect it:

git remote add origin https://github.com/your-username/your-repo.git
🚀 6. Push to GitHub

Rename your branch to main and push:

git branch -M main
git push -u origin main

Use the correct remote name: origin (not orgin).

🔁 7. Common Git Commands 
git status            # View current changes
git add filename.ext  # Stage specific file
git commit -m "msg"   # Commit with message
git pull              # Fetch changes from GitHub
git push              # Push your changes
🧠 8. Check or Fix Remote URL 
git remote -v         # View current remote URL

# If incorrect:
git remote set-url origin https://github.com/your-username/your-repo.git
✅ 9. Final Notes

Make sure to:

  • Use HTTPS or SSH correctly
  • Check remote URL spelling
  • Use correct branch (main or master)
  • Generate a Personal Access Token if required for HTTPS

🎉 You’ve successfully pushed code to GitHub!

📂 11. Cloning a Repository

Use this to copy a GitHub repository to your local machine:

git clone https://github.com/username/repo-name.git
👥 12. Branching and Merging

Work on different features without disturbing the main branch.

git branch feature-branch     # Create a branch

git checkout feature-branch  # Switch to it

# Or create and switch in one

git checkout -b new-feature

git merge feature-branch     # Merge into current branch
🔄 13. Resolving Merge Conflicts

If two branches edit the same line, Git will ask you to fix the conflict manually.

# After fixing files:
git add .
git commit -m "Fix merge conflict"
🧰 14. Stashing Changes

Save uncommitted changes for later without committing.

git stash          # Stash changes

# Later

git stash apply   # Reapply stashed changes
📊 15. Viewing Commit History

Track the history of changes:

git log            # Full history

git log --oneline # Simplified view
🔐 16. GitHub SSH Authentication

Set up SSH for secure authentication:

ssh-keygen -t ed25519 -C "your_email@example.com"

# Add key to SSH agent

ssh-add ~/.ssh/id_ed25519

Add the public key to your GitHub account under Settings → SSH and GPG keys.

📦 17. Fork vs Clone
  • Fork: Creates a personal copy of a GitHub repo on your account. Ideal for contributing.
  • Clone: Copies a repo to your local machine.
🌐 18. GitHub Pages for Hosting

You can host static websites from a GitHub repo:

  1. Create a public repo
  2. Add HTML/CSS/JS files
  3. Go to repo Settings → Pages
  4. Select branch and save. URL will be https://your-username.github.io/repo-name
🧑‍🤝‍🧑 19. Collaborators and Permissions

In private/public repos, you can invite collaborators with different roles:

  • Admin: Full access
  • Write: Push changes
  • Read: View-only
🚨 20. Revert and Reset

If things go wrong:

git checkout filename        # Discard changes in a file

git reset --hard HEAD       # Reset to last commit

git revert COMMIT_ID        # Create new commit that undoes
📘 21. Glossary
  • Commit: Save changes
  • Branch: Parallel version of repo
  • Merge: Combine branches
  • Clone: Copy repo to local machine
  • Fork: Copy repo to GitHub account
  • Pull Request: Propose code change
  • Push: Upload changes
  • Pull: Download changes
